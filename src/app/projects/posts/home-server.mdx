A self-hosted private cloud platform built on Ubuntu, Docker, Nextcloud, Cloudflare Tunnel, and Tailscale, providing secure remote file storage, backup, and media sharing capabilities.

> **Status:** Operational  
> **Role:** System Administration, Infrastructure Design, Networking, Security Configuration  
> **Technologies:** Linux, Ubuntu Server, OpenSSH, Docker, Nextcloud, Cloudflare Tunnel, Tailscale, Google Authenticator  

---

## Project Summary

This project was developed to create a private cloud storage platform capable of securely storing, backing up, and sharing files without relying on subscription-based cloud storage providers.

The system is hosted on a Dell OptiPlex 3060 Micro running Ubuntu and utilizes Docker containers to deploy and manage a Nextcloud instance. Secure remote access is provided through a Cloudflare Tunnel connected to a custom domain, while administrative access is performed through SSH over a private Tailscale VPN network with multi-factor authentication enabled.

Beyond its practical use as a storage platform, the project served as an opportunity to gain hands-on experience with Linux administration, containerized services, networking, authentication systems, and self-hosted infrastructure deployment.

## System Architecture

The system consists of five primary components:

1. **Ubuntu Server** Hosts all services
2. **Docker Environment** Manages containerized applications
3. **Nextcloud Container** Provides cloud storage functionality
4. **Cloudflare Tunnel** Enables secure remote user access
5. **Tailscale Network** Provides secure administrative access

## Secure Remote Access

One of the primary design challenges was balancing security with ease of use.

**Cloudflare Tunnel**

Initial testing relied exclusively on Tailscale VPN access. While secure, this approach required every user to install and configure Tailscale before accessing files.

To simplify access, a custom domain was purchased through Cloudflare and connected to the server using a Cloudflare Tunnel.

Benefits include:

- No port forwarding required
- Simplified access for non-technical users
- Encrypted connections
- Reduced exposure of internal network infrastructure

Users can now access the platform directly through a web browser using the configured domain name.

---

**Tailscale VPN**

Administrative access remains restricted to a private Tailscale mesh network.

Benefits include:

- Private device-to-device connectivity
- Secure SSH access
- Simplified remote administration

This architecture separates administrative traffic from public user traffic, improving overall system security.

---

**Multi-Factor Authentication**

Google Authenticator was integrated into the SSH authentication workflow to improve server security.

Authentication requires:

1. Username
2. Password
3. Time-based authentication code

This provides an additional security layer beyond password-only authentication.

## Problems Faced

**Ubuntu Installation Issues**

During initial deployment, Ubuntu installation repeatedly completed successfully but failed to boot afterward, resulting in a "No Bootable Device" error.

After troubleshooting BIOS settings and installation media, the issue was traced to the USB port being used during installation. Moving the installation drive from a front-panel USB port to a rear motherboard-connected USB port resolved the problem and allowed Ubuntu to boot normally.

---

**SSH Authentication Lockout**

While configuring Google Authenticator integration, SSH access was accidentally disabled due to an incorrect authentication configuration.

Because remote access was no longer available, direct access to the server was required using a monitor, keyboard, and mouse.

The issue was ultimately traced to authentication settings within the SSH configuration files. After restoring access and carefully reconfiguring the authentication workflow, multi-factor authentication was successfully implemented without disrupting SSH connectivity.

---

**Remote User Accessibility**

The original architecture relied entirely on Tailscale VPN access for all users.

Although secure, this created onboarding challenges for family members and non-technical users.

The final solution utilized a Cloudflare Tunnel and custom domain, allowing users to connect directly through a web browser while retaining secure administrative access through Tailscale.
